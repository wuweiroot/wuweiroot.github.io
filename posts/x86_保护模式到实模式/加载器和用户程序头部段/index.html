    加载器的工作流程:#

    • 读取用户程序的起始扇区
    • 把整个用户程序都读入内存 ，需要确认用户程序加载位置
    • 计算段的物理地址和逻辑段地址（段的重定位）
    • 转移用户程序执行（将处理器的控制权交给用户程序）

    读取用户程序的起始扇区#

    IO 设备#

    些设备分成两种，一种是输入设备，比如键盘、鼠标、 麦克风、摄像头等；另一种是输出设备，比如显示器、打印机、扬声器等。输入设备和输出设备统 称输入输出（Input/Output，I/O）设备

    不同的外围设备，都有各自不同的 I/O接口

    IO 接口#

    I/O 接口可以是一个电路板，也可能是一块小芯片，这取决于它有多复杂。无论如何，它是一 个典型的变换器，或者说是一个翻译器，在一边，它按处理器的信号规程工作，负责把处理器的信 号转换成外围设备能接受的另一种信号；在另一边，它也做同样的工作，把外围设备的信号变换成 处理器可以接受的形式。

    IO 指令#

    在 Intel 的系统中，只允许 65536（十进制数）个端口存在，端口号从 0 到 65535（0x0000～0xffff）。 因为是独立编址，所以，端口的访问不能使用类似于 mov 这样的指令，取而代之的是 in 和 out 指令
    IN指令： 端口到CPU in ax,端口号
    OUT指令， CPU到端口 out 端口号，ax

    I0 Conterller Hub : ICH#

    输入输出控制设备集中器（I/O Controller Hub，ICH）芯片，该芯 片的作用是连接不同的总线，并协调各个 I/O 接口对处理器的访问。在个人计算机上，这块芯片就 是所谓的南桥。

    IO 端口#

    端口是处理器和外围设备通过 I/O 接口交流的窗口，每一个 I/O 接口都可能拥有好几个端口， 分别用于不同的目的。

    硬盘–块（Block）设备#

    “CHS” 是 “Cylinder-Head-Sector” 的缩写，表示硬盘的物理几何结构。在早期的硬盘中，数据的寻址是通过柱面（Cylinder）、磁头（Head）和扇区（Sector）来实现的。这种寻址方式被称为 CHS 寻址。然而，随着技术的发展，CHS 寻址方式逐渐被 LBA 寻址方式所取代。 “LBA” 是 “Logical Block Addressing” 的缩写，表示逻辑块寻址。LBA 是一种更为直观和简单的硬盘寻址方式，它将硬盘的扇区抽象为一个线性的逻辑块序列，从0开始编号。使用 LBA 寻址方式，可以更方便地寻址硬盘上的数据，而不需要关心物理结构。

    硬盘端口，1f0~1f7#

    数据寄存器端口：通常是 0x1F0 或 0x170。这个端口用于读取或写入硬盘的数据。 错误寄存器端口：通常是 0x1F1 或 0x171。这个端口用于读取硬盘的错误信息。 扇区计数寄存器端口：通常是 0x1F2 或 0x172。这个端口用于指定要读取或写入的扇区数。 扇区号寄存器端口：通常是 0x1F3 或 0x173。这个端口用于指定要读取或写入的起始扇区号。 柱面低 8 位寄存器端口：通常是 0x1F4 或 0x174。这个端口用于指定要读取或写入的柱面号的低 8 位。 柱面高 8 位寄存器端口：通常是 0x1F5 或 0x175。这个端口用于指定要读取或写入的柱面号的高 8 位。 驱动器/头寄存器端口：通常是 0x1F6 或 0x176。这个端口用于指定要访问的硬盘驱动器和磁头号。 状态寄存器端口：通常是 0x1F7 或 0x177。这个端口用于读取硬盘的状态信息。

    read_hard_disk_0:                        ;从硬盘读取一个逻辑扇区
                                         ;输入：DI:SI=起始逻辑扇区号  xxxx xxxx xxxx xxxx xxxx xxxx xxxx xxxx
                                         ;      DS:BX=目标缓冲区地址
         push ax
         push bx
         push cx
         push dx
      
         ;读取的扇区数
         mov dx,0x1f2                    ;扇区计数寄存器端口
         mov al,1
         out dx,al                       ;读取的扇区数

         inc dx                          ;0x1f3 加一指令
         mov ax,si
         out dx,al                       ;LBA28地址7~0

         inc dx                          ;0x1f4
         mov al,ah
         out dx,al                       ;LBA地址15~8

         inc dx                          ;0x1f5
         mov ax,di
         out dx,al                       ;LBA地址23~16

         inc dx                          ;0x1f6
         mov al,0xe0                     ;LBA28模式，主盘
         or al,ah                        ;LBA地址27~24
         out dx,al

         inc dx                          ;0x1f7
         mov al,0x20                     ;读命令
         out dx,al

  .waits:
         in al,dx                         ;读取端口 1f7
         and al,0x88
         cmp al,0x08
         jnz .waits                      ;比较失败跳转，比较成功继续，代表 不忙，且硬盘已准备好数据传输 

         mov cx,256                      ;总共要读取的字数 ，一个扇区是256字节
         mov dx,0x1f0
  .readw:
         in ax,dx                        ;端口1f0的数据传给AX
         mov [bx],ax                      ;ax中数据写入BX指向的内存
         add bx,2                         ;指针偏移
         loop .readw

         pop dx
         pop cx
         pop bx
         pop ax
      
         ret

    ;确认用户程序加载位置，phy_base 0x10000 转换为逻辑地址（段地址 ds +偏移地址 0）#

    mov ax,[cs:phy_base]            ;计算用于加载用户程序的逻辑段地址 
mov dx,[cs:phy_base+0x02]
mov bx,16        
div bx     ;DIV src     如果 src 是一个字（16位）或者一个双字，则被除数由 DX:AX 组成，商将存储在 AX 中，余数将存储在 DX 中。
mov ds,ax                 ;令DS和ES指向该段（用户程序目的地址，0x10000）以进行操作
mov es,ax

    过程调回#

    16 位相对近调用#

    位相对近调用是三字节指令，操作码为 0xE8，后跟 16 位的操作数，因为是相对调用，故该 操作数是当前 call 指令相对于目标过程的偏移量。

    call proc_1 (call near proc_1)

    它用指令指针寄存器 IP 的当前内容加上指令中的操作数，再加上 3，得到一个新的偏移地址。接着，将 IP 的 原有内容压入堆栈。最后，用刚才计算出的偏移地址取代 IP 原有的内容。这直接导致处理器的执 行流转移到目标位置处

    16 位间接绝对近调用#

    这种调用也是近调用，只能调用当前代码段内的过程，指令 中的操作数不是偏移量，而是被调用过程的真实偏移地址，故称为绝对地址。不过，这个偏移地址 不是直接出现在指令中，而是由 16 位的通用寄存器或者 16 位的内存单元给出。

    call cx ;目标地址在 CX 中。省略了关键字“near”，下同
    call [0x3000] ;要先访问内存才能取得目标偏移地址
    call [bx] ;要先访问内存才能取得目标偏移地址
    call [bx+si+0x02] ;要先访问内存才能取得目标偏移地址

    16 位直接绝对远调用#

    这种调用属于段间调用，即调用另一个代码段内的过程，所 以称为远调用（Far Call）。很容易想到，远调用既需要被调用过程所在的段地址，也需要该过程在 段内的偏移地址
    16 位”是针对偏移地址来说的，而不用于限定段地址；“直接”的意思是，段地址和偏移地址 直接在 call 指令中给出了。当然，这里的地址也是绝对地址。

    call 0x2000:0x0030 //0x9A 是操作码，后面跟着的两个字分别是偏移地址和段地址，按规定，偏移地址在前，段地址在后 call far [0x2000]
    call far [proc_1] call far [bx] //访问 dx:bx 内存中的两个字，偏移地址和段地址,放入IP和CS call far [bx+si]

    将代码段寄存器 CS 和指令指针寄存器 IP 的当前内容分别压栈；最后，用刚才取得的段地址和偏移地址分别取代 CS 和 IP 的原值
