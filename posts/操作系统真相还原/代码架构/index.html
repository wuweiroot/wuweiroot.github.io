    内存布局#

    低端1M可用内存#

    结论是在 0x7e00～0x9fbff 这片区域的高地址中找一亩地给kernel.bin，这里我擅自做主啦，帮大家选的是 0x70000。

    流程#

    bios -> mbr - >loder ->kernel

    • BIOS将第一扇区（mbr）加载到0x7c00内存位置后跳转的0x7c00处执行。
    • mbr读取第二、三、四、五扇区(loder)加载到0x900内存位置后跳转到 0x900 +0x300处执行
      • 0x500～0x7BFF 和 0x7E00～9FBFF 这两段内存区域可用。
      • loader 放在低处，多留出一些空间给内核
      • 将 loader 的加载地址选为 0x900。为什么不是 0x500，这个多省空间？还是预留出一定空间吧 ， 彼此隔开远一点心里才踏实
    • loder创建GDT表，进入保护模式，加载内核（内核位于第9扇区）到 0x70000，创建页目录 ，初始化内核，进入内核
      • 随意取的，是觉得 0x70000～0x9fbff 有 0x2fbff=190KB 字节的空间，而我们的内核不超过 100KB，

    boot#

    mbr#

    mbr读取第二、三、四、五扇区(loder)加载到0x900内存位置后跳转到 0x900 + 0x300处执行

    loder#

    • 位于硬盘第二扇区，被加载到0x500内存位置
    • 读取内存布局存在标号total_mem_bytes的内存位置

    进入保护模式#

    - 打开A20
- 加载GDT ，lgdt [gdt_ptr]
- cr0第0位置1，打开分页
    • 跳转，刷新流水线
      • jmp SELECTOR_CODE:p_mode_start
        • 值的更新：CS 寄存器会被设置为 SELECTOR_CODE。
        • 段描述符加载：CPU 会根据 CS 中的段选择子从 GDT 中找到对应的代码段描述符
        • EIP 寄存器会被设置为 p_mode_start
    • 初始化ds es ss寄存器， sp堆栈指针，gs寄存器（显存指针）
    • 加载内核（内核位于第9扇区）到 0x70000

    创建页目录和页表#

    • 内核地址映射到虚拟地址3G以上
      • 虚拟地址0xc0000000以上的1MB地址映射到物理地址1MB之内
      • 物理地址范围 0～0x3fffff 之间的物理页，这也就是虚拟地址 0x0～0x3fffff和虚拟地址 0xc0000000～0xc03fffff 对应的物理页
    • 页目录表在内存中的起始位置——从1M开始的位置
    • 1M开始的4KB置为0，将页目录表初始化
      • 第一个页表位置位于也目录上方，页目录位置+0x1000(4kb)
      • 第一个页表位置写入第0号和第768号(地址偏移0xc00)页目录项
        • 0xc0000000 的高 10 位是 0x300，即十进制的 768
      • 页目录自身地址写入最后一个页目录项
      • 紧挨着的第二个页表第三个到第254个页表地址依次写入页目录项的796-1026号位置
    • 初始化第一个页表
      • 将低端地址1M(0x1000,4096)空间的256个物理页地址依次填写到页表项中
        • 4kb=0x1000=4*1024=4096 1Mb= 0x100000

    • 更新GDT描述符中视频段描述符的内容
    • 将gdt的基址加上0xc0000000使其成为内核所在的高地址
    • 在开启分页后,用gdt新的地址重新加载

    进入内核#

    - 初始化内核
    - 通过elf格式解析kernel.bin
- 跳转到内核入口虚拟地址 0xc0001500
    - 编译时候指定入口虚拟地址为0xc0001500

    标号定义#

    • total_mem_bytes 标号位于文件头偏移0x200位置用来存储内存结构
    • loader_start 标号位于文件头偏移0x300位置，是loder程序入口

    GDT表数据定义#

    1. 构建gdt及其内部的描述符,第一个全局描述符表设置为空不使用
    GDT_BASE:                                               
    dd 0x00000000 
	dd 0x00000000

CODE_DESC:  
    dd 0x0000FFFF 
	dd DESC_CODE_HIGH4

DATA_STACK_DESC:  
    dd 0x0000FFFF
    dd DESC_DATA_HIGH4

VIDEO_DESC: 
    dd 0x80000007	                                    ;limit=(0xbffff-0xb8000)/4k=0x7
    dd DESC_VIDEO_HIGH4                                 ; 此时dpl已改为0
    1. 段选择子数据构造
    SELECTOR_CODE equ (0x0001<<3) + TI_GDT + RPL0       ; 相当于(CODE_DESC - GDT_BASE)/8 + TI_GDT + RPL0
SELECTOR_DATA equ (0x0002<<3) + TI_GDT + RPL0	    ; 同上
SELECTOR_VIDEO equ (0x0003<<3) + TI_GDT + RPL0	    ; 同上
    1. gdt寄存器加载数据构造
    gdt_ptr dw GDT_LIMIT                                    ;定义加载进入GDTR的数据，前2字节是gdt界限，后4字节是gdt起始地址，
	    dd  GDT_BASE

    kernel\memory.c#

    内存管理

    • PDE_IDX

      • #define PDE_IDX(addr) ((addr & 0xffc00000) » 22)
      • 虚拟地址高10位，PDE（Page Directory Entry）页目录条目索引

    • PTE_IDX

      • #define PTE_IDX(addr) ((addr & 0x003ff000) » 12)
      • 虚拟地址中10位，PTE（Page Table Entry ）页表条目索引

    • pde_ptr

      • 构造虚拟地址vaddr对应的pde指针
      • pde_ptr经过CPU的段部件，页部件处理后得到PDE地址
      uint32_t* pde_ptr(uint32_t vaddr) {
    /* 0xfffff是用来访问到页表本身所在的地址 */
    uint32_t* pde = (uint32_t*)((0xfffff000) + PDE_IDX(vaddr) * 4);
    return pde;
}

    • pte_ptr

      • 构造虚拟地址vaddr对应的pte指针
      • pte_ptr经过CPU的段部件，页部件处理后得到PTE地址
      uint32_t* pte_ptr(uint32_t vaddr) {
    /* 先访问到页表自己 + 再用页目录项pde(页目录内页表的索引)做为pte的索引访问到页表 + 再用pte的索引做为页内偏移*/
    uint32_t* pte = (uint32_t*)(0xffc00000 + ((vaddr & 0xffc00000) >> 10) + PTE_IDX(vaddr) * 4);
    return pte;
}

    • page_table_add

      • 页表中添加数据，完成虚拟地址物理地址映射
        • 通过pte_ptr的到虚拟地址的页表指针
        • 将物理地址写入页表指针指向的地址。

    • mem_pool_init

    传入从BIOS获取的可用内存大小,初始化用来内存管理的位图

    
#define MEM_BITMAP_BASE 0xc009a000

- used_mem = page_table_size + 0x100000;	  // 已使用内存 = 1MB + 256个页表
- free_mem = all_mem - used_mem;    //剩余可用内存
- all_free_pages = free_mem / PG_SIZE; //剩余可用内存页数量
- kernel_free_pages = all_free_pages / 2 ， user_free_pages = all_free_pages - kernel_free_pages;  //一半分给用户一半给内核

-  kbm_length = kernel_free_pages / 8;			  // 内核物理内存池的位图长度,位图中的一位表示一页,以字节为单位
-  ubm_length = user_free_pages / 8;			  // 用户物理内存池的位图长度.
    - 一个bit表示一个字节，一个bit表示一页

-  kp_start = used_mem;				  // Kernel Pool start,内核使用的物理内存池的起始地址
-  up_start = kp_start + kernel_free_pages * PG_SIZE;	  // User Pool start,用户使用的物理内存池的起始地址



kernel_pool.phy_addr_start = kp_start;       //赋值给内核使用的物理内存池的起始地址
user_pool.phy_addr_start   = up_start;       //赋值给用户使用的物理内存池的起始地址

kernel_pool.pool_size = kernel_free_pages * PG_SIZE;     //赋值给内核使用的物理内存池的总大小
user_pool.pool_size	 = user_free_pages * PG_SIZE;       //赋值给用户使用的物理内存池的总大小

kernel_pool.pool_bitmap.btmp_bytes_len = kbm_length;     //赋值给管理内核使用的物理内存池的位图长度
user_pool.pool_bitmap.btmp_bytes_len	  = ubm_length;   //赋值给管理用户使用的物理内存池的位图长度

// 32M内存占用的位图是2k.内核内存池的位图先定在MEM_BITMAP_BASE(0xc009a000)处.
   	kernel_pool.pool_bitmap.bits = (void*)MEM_BITMAP_BASE;      //管理内核使用的物理内存池的位图起始地址
							       
/* 用户内存池的位图紧跟在内核内存池位图之后 */
   	user_pool.pool_bitmap.bits = (void*)(MEM_BITMAP_BASE + kbm_length);     //管理用户使用的物理内存池的位图起始地址

    device\keyboard.c#

    键盘驱动

    • 键盘扫描码，每个案件对应的数值
      • 扫描码由按下的码（通码），弹起来的吗（断码）组成
        • 第一套扫描码的通码断码是一个字节，断码=0x80+通码
          • 第二套扫描码中的断码是在通码的前面加0xf0一共两个字节
          • 8042将第二套通码和断码转换为第一套通码和断码
        • -第一套最高位的值表示按键状态
      • 有些后下的扩展按键前面加有一个字节前缀比如0xe0
    • 按键或者松开会触发一次中断。

    键盘扫描码有三套#

    • 兼容设计，将其他套键盘扫描码转换为第一套的扫描码

    int printf(const char *format, …)#

    示例：printf(“cat: open: open %s failed\n”, argv[1]);

    • ap 指向函数调用参数栈中第一个参数的地址。
    • 遍历入参第一个字符串内%的位置得到类型，通过va_arg(ap, t)拿到对应参数的值，将这个值放入buf
    • 其他地方的字符原样复制到buf
    #define va_start(ap, v) ap = (va_list) & v // 把ap指向第一个固定参数v
#define va_arg(ap, t) *((t *)(ap += 4))    // ap指向下一个参数并返回其值
#define va_end(ap) ap = NULL               // 清除ap

uint32_t printf(const char *format, ...)
{
    va_list args;
    va_start(args, format); // 使args指向format
    char buf[1024] = {0};   // 用于存储拼接后的字符串
    vsprintf(buf, format, args);
    va_end(args);
    return write(1, buf, strlen(buf));
}

/* 将参数ap按照格式format输出到字符串str,并返回替换后str长度 */
uint32_t vsprintf(char *str, const char *format, va_list ap)
{
    char *buf_ptr = str;
    const char *index_ptr = format;
    char index_char = *index_ptr;
    int32_t arg_int;
    char *arg_str;
    while (index_char)
    {
        if (index_char != '%')
        {
            *(buf_ptr++) = index_char;
            index_char = *(++index_ptr);
            continue;
        }
        index_char = *(++index_ptr); // 得到%后面的字符
        switch (index_char)
        {
        case 's':
            arg_str = va_arg(ap, char *);
            strcpy(buf_ptr, arg_str);
            buf_ptr += strlen(arg_str);
            index_char = *(++index_ptr);
            break;
        case 'c':
            *(buf_ptr++) = va_arg(ap, char);
            index_char = *(++index_ptr);
            break;
        case 'd':
            arg_int = va_arg(ap, int);
            if (arg_int < 0)
            {
                arg_int = 0 - arg_int; /* 若是负数, 将其转为正数后,再正数前面输出个负号'-'. */
                *buf_ptr++ = '-';
            }
            itoa(arg_int, &buf_ptr, 10);
            index_char = *(++index_ptr);
            break;
        case 'x':
            arg_int = va_arg(ap, int);
            itoa(arg_int, &buf_ptr, 16);
            index_char = *(++index_ptr); // 跳过格式字符并更新index_char
            break;
        }
    }
    return strlen(str);
}

    系统调用#

    #define _syscall2(NUMBER, ARG1, ARG2) ({    \
    int retval;                             \
    asm volatile(                           \
        "int $0x80"                         \
        : "=a"(retval)                      \
        : "a"(NUMBER), "b"(ARG1), "c"(ARG2) \
        : "memory");                        \
    retval;                                 \
})


核心原理
中断触发：int $0x80 指令触发软件中断，CPU 切换到内核态，执行系统调用处理程序。
参数传递：
eax：系统调用号（NUMBER）。
ebx：第一个参数（ARG1）。
ecx：第二个参数（ARG2）。
若有更多参数，依次使用 edx（第 3 个）、esi（第 4 个）、edi（第 5 个）、ebp（第 6 个）。
返回值：eax 寄存器存储系统调用结果（成功时为非负数值，失败时为 -1，错误码存入 errno）
