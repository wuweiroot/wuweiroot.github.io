    计算机通识

    访问外设的两种方式#

    1. 内存映射： 通过地址总线将外设自己的内存映射到某个内存区域（并不是映射到主板上插的内存条中）。
    2. 端口操作：外设都有自己的控制器，控制器上有寄存器，这些寄存器就是所谓的端口，通过 in/out 指令读写端口来访问硬件的内存。

    中断向量表#

    • CPU 收到中断后，得知道发生了什么事情才能执行相应的处理办法。这是通过中断向量表或中断描述 符表（中断向量表是实模式下的中断处理程序数组，在保护模式下已经被中断描述符表代替，

    • 而 Linux 内核是在进入保护模式后才建立中断例程的，不过在保护模式下，中断向量表已经不存在了， 取而代之的是中断描述符表（Interrupt Descriptor Table， IDT）。

    • 如果在实模式下执行 int 指令，会自动去访问中断向量表。如果在保护模式下执行 int 指令，则会自动访问中断描述符表。

    LEGACY#

    Legacy BIOS：IDT 由操作系统在引导阶段创建，BIOS 仅提供实模式的中断向量表

    BIOS 的角色#

    • BIOS 在实模式下提供中断向量表（IVT），其固定地址为 0x00000，包含实模式下的中断服务程序入口（如 INT 10h 显示服务、INT 13h 磁盘服务等）8910。

    • BIOS 不涉及保护模式的中断管理，仅负责实模式下的硬件初始化和基本中断支持。

    操作系统的职责#

    • 当系统从实模式切换到保护模式时，引导加载程序（如 GRUB） 或操作系统内核需手动设置 IDT，包括定义中断门、陷阱门等描述符，并通过 LIDT 指令加载 IDT 的基地址和界限到 IDTR 寄存器56。

    • 例如，Linux 内核的 setup.s 代码在引导阶段初始化 IDT 和 GDT，为保护模式的中断机制做准备

    UEFI#

    UEFI：UEFI 固件可能初始化临时 IDT，但最终由操作系统内核接管并重建完整的 IDT

    UEFI 固件的角色#

    • UEFI 原生运行在保护模式（或长模式），其固件已初始化基础的内存管理和中断框架，可能包含临时 IDT 用于 UEFI 自身的服务（如安全启动、硬件检测等）1。

    • UEFI 提供启动服务（EFI_BOOT_SERVICES），允许操作系统调用固件接口获取硬件信息，但不直接暴露 IDT 的配置细节。

    操作系统的职责#

    • 操作系统内核在接管控制权后（退出 UEFI 启动服务环境时），需重新初始化自己的 IDT，以适配内核的中断处理逻辑（例如定义系统调用、硬件异常等）

    • UEFI 模式下，操作系统无需手动开启 A20 地址线或设置 GDT（UEFI 已提供），但 IDT 仍需由内核根据自身需求定制。

    关键区别对比#

    模式 IDT 创建主体 中断管理阶段 依赖的固件功能
    Legacy BIOS 操作系统（引导加载程序/内核） 保护模式切换后 仅实模式 IVT，无 IDT 支持
    UEFI 操作系统（内核） 内核初始化阶段 UEFI 提供临时中断框架
