    gdb调试命令

    March 13, 2026

    调试步骤#

    从机#

    配置gdb调试模式，启动

    主机#

    在gdb窗口中输入命令：target remote localhost:1234 以连接bochs进行调试

    然后输入命令：symbol-file /home/lmx/Desktop/the_truth_of_operationg_system/chapter_15/k/build/kernel.sym 以加载kernel.bin的符号表

    编译#

    • 包含gdb调试

      • CFLAGS= -Wall $(LIB) -c -fno-builtin -W -Wstrict-prototypes -Wmissing-prototypes -m32 -fno-stack-protector -g

    • 生成可以被GDB理解的符号表，用于GDB调试

      • gdb_symbol:
        • objcopy –only-keep-debug $(BUILD_DIR)/kernel.bin $(BUILD_DIR)/kernel.sym

    部署#

    • file 命令用于指定要调试的可执行文件，会加载文件的全部内容，包括代码、数据和调试符号（如果有的话）。
    • symbol-file 命令用于加载单独的符号文件，只包含调试符号信息，适合在可执行文件和调试符号分离的情况下使用。
    • 通常在调试时，你可以先用 file 命令加载可执行文件，再用 symbol-file 命令加载单独的符号文件，这样既能保证可执行文件的简洁，又能在调试时获得完整的符号信息。

    加载精简符号表#

    分离调试符号：正如之前提到的使用 objcopy –only-keep-debug 命令将调试符号从 kernel.bin 中提取出来，保存到单独的文件（如 kernel.sym）中。这样，kernel.bin 文件体积会减小，便于部署，而调试时可以通过调试工具（如 GDB）加载 kernel.sym 文件来获取调试信息。

    symbol-file /home/lmx/Desktop/the_truth_of_operationg_system/chapter_15/k/build/kernel.sym

    加载整个文件#

    symbol-file /home/lmx/Desktop/the_truth_of_operationg_system/chapter_15/k/build/kernel.bin

    断点#

    b main.c:60

    查看断点#

    info breakpoints

    常用控制#

    run（r）：启动被调试程序，若程序需要参数，可在 run 后直接添加。例如 run arg1 arg2。 continue（c）：继续执行被调试程序，直到遇到下一个断点或程序结束。 next（n）：单步执行程序，不进入函数内部，直接执行下一条语句。 step（s）：单步执行程序，会进入函数内部。

    进入反汇编#

    layout asm

    nexti（ni）：单步执行下一条汇编指令，但不进入函数调用。 stepi（si）：单步执行下一条汇编指令，如果是函数调用则进入函数内部。

    退出反汇编#

    layout prev 可用于切换到上一个布局
