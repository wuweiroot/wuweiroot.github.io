    git常用命令#

    初始化仓库#

    • cd public
    • git init ##初始化仓库
    • git remote add origin https://github.com/caecarxu/caecarxu.github.io.git ##链接远程仓库
    • git add .
    • git commit -m “first commit”
    • git push -u origin master

    提交代码#

    • cd public
    • git add .
    • git status
    • git commit -m “add blog post”
    • git push

    用SSH格式的远程地址#

    • 先查看当前远程地址（确认是 HTTPS 格式） git remote -v
    • 删除旧的 HTTPS 远程地址 git remote rm origin
    • 添加 SSH 格式的远程地址（替换成你的仓库） git remote add origin git@github.com:wuweiroot/wuweiroot.github.io.git
    • 再次推送 git push -u origin master

    和远程仓库建立链接#

    git push –set-upstream origin master

    简写形式（效果完全一样，更常用）：#

    git push -u origin master#

    强制推送本地 master 分支到远程，建立追踪关系#

    git push -uf origin master

    git cherry-pick#

    一个分支的commit到另外一个分支

    1. 先查看源分支（比如 dev）的提交记录，找到要复制的 commit ID（前 7 位即可） git log dev –oneline # 输出示例：a1b2c3d fix: 修复登录bug | 987654e feat: 新增支付功能
    2. 切换到目标分支（比如 master） git checkout master
    3. 挑拣指定提交到当前分支（替换成你的 commit ID） git cherry-pick a1b2c3d
    4. 若挑拣有冲突：解决冲突 → 标记 → 继续挑拣 git add . git cherry-pick –continue # 完成挑拣
    • （若想放弃挑拣：git cherry-pick –abort）
    1. 推送远程（可选） git push origin master

    git rebase#

    更改分支代码拉取节点

    1. 切到 master，拉取最新代码 git checkout master git pull origin master

    2. 切回 dev 分支 git checkout dev

    3. 执行 rebase，把 dev 基于 master 最新提交重排 git rebase master

    4. 假设出现冲突： ① 打开冲突文件，删除 «««< HEAD 等标记，保留正确代码 ② 标记冲突已解决 git add . ③ 继续 rebase git rebase –continue

    5. rebase 完成后，推送 dev 分支（强制推送，仅个人分支！） git push -f origin dev

    6. 此时 dev 已同步 master 最新代码，可合并到 master（无冲突） git checkout master git merge dev git push origin master

    git stash#

    暂存本地代码

    暂存工作区所有未提交的修改（已追踪文件的修改+新增文件）#

    git stash

    推荐：给暂存加备注，方便后续识别（多人协作/多暂存时必备）#

    git stash save “修复订单页bug：改了一半的代码”

    恢复最新的暂存（stash@{0}）到工作区#

    git stash apply

    恢复指定的暂存（比如 stash@{1}）#

    git stash apply stash@{1}
